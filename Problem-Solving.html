Problem Solving.

What was one time you got blocked on a simple problem?

During the "Manipulating JS Objects" section of the Kata, I couldn't seem to figure out how on earth I was supposed to make Carson (an object) one of Terah's (another object) children (an object key) . I was trying to put an object inside an object which was inside another object.

First of all I tried... something, anything and wanted to see if I could get it to work. So, I wrote:

terah.children.child1 = carson

I did this because I wanted to list Carson as one of the Children of Terah and so I figured I'd just have the key "child1" as Carson's value (thats my mistake but I didn't know it yet).

I tried googling it for a while and found various questions and guides on similar topics but different enough to leave me confused as to how I would take the same approach. I sat there googling and staring for about 30-40 minutes.

I tried reading the error message which went:

8. false: carson should be defined as an object and assigned as a child of Terah
 Error: carson should be defined as an object and assigned as a child of Terah
at assert (/home/runner/Manipulating-JS-Objects/index.js:49:11)
at /home/runner/Manipulating-JS-Objects/index.js:97:1
at Script.runInContext (vm.js:131:20)
at Object.<anonymous> (/run_dir/interp.js:156:20)
at Module._compile (internal/modules/cjs/loader.js:1133:30)
at Object.Module._extensions..js (internal/modules/cjs/loader.js:1153:10)
at Module.load (internal/modules/cjs/loader.js:977:32)
at Function.Module._load (internal/modules/cjs/loader.js:877:14)
at Function.executeUserEntryPoint [as runMain](internal/modules/run_main.js:74:12)

While I understood its issue listed, (Carson should be defined as an object) I was baffled because earlier I had Carson defined as an object.

Eventually, I got tired. Frustrated, I decided to finish for the day and come back to it tomorrow. Which is when things went better for me, I woke up, meditated and had an epiphany as to what I was doing wrong, that I needed to place Carson as the key and that I was adding an extra value where it wasn't needed.

This felt strange as an experience, I had made an assumption that I was still too new to this to realise my own mistakes. That sounds like a silly presumption but it was how I felt. I guess coming through this I had to challenge my own assumptions as to how capable I am with any new skills I develop here. If I learn something, own it and be proud of it. Hell yeah, at this stage, I can write html, css and some Javascript and I'm only gonna get better at it.
